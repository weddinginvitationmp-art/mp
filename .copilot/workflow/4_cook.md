# Phase: /cook (Implementation)

## Mục tiêu
Triển khai giải pháp theo thiết kế đã định nghĩa trong `/arch`.

## Nguyên tắc
- Tuân thủ SOLID principles.
- Code phải self-documenting và dễ đọc.
- Viết unit test song song; TDD được khuyến khích.
- Giữ hàm nhỏ, tập trung, tránh duplication.
- Commit hoặc ghi rõ patch từng bước nếu cần.

## Output
- Code và test cases tương ứng.
- Nếu cần, cập nhật tài liệu nội bộ của module.

## Ghi chú
- Nếu task là design/analysis, vẫn cần output rõ ràng thay vì code.
- Với code thay đổi, chạy test sớm và thường xuyên.
- Nếu có multiple implementation options, chọn option đơn giản nhất thỏa yêu cầu.
