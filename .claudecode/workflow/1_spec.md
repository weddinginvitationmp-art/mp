# Phase: /spec (Specification)

## Mục tiêu
Hiểu chính xác "Cái gì", "Tại sao", và giới hạn của giải pháp.

## Yêu cầu đầu ra
- Liệt kê các User Stories.
- Xác định Functional và Non-functional requirements.
- Định nghĩa acceptance criteria và success criteria.
- Đưa ra assumptions, non-goals, và open questions.
- Xác định các ràng buộc kỹ thuật (tech stack hiện tại) và dependencies.
- Xác nhận scope với user.

## Output
- `DOCS/SPEC/{unique name + short summary spec + datetime}`
- Ở bên trong sẽ chứa nhiều file .md thể hiện cho từng phase spec

## Ghi chú
- Không viết code trong bước này.
- Nếu yêu cầu chưa rõ, hỏi thêm questions để làm rõ scope và edge cases.
